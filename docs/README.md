# Internal Documentation

This directory contains **internal documentation** for the E2E Testing Agent development team.

## Public Documentation

For user-facing documentation, visit: **[docs.heyargus.ai](https://docs.heyargus.ai)**

Public docs are maintained in the [argus-docs](https://github.com/RaphaEnterprises-AI/argus-docs) repository.

## Internal Documentation Structure

### Architecture (`architecture/`)
Internal implementation specifications and design documents:
- **ARCHITECTURE_ASSESSMENT.md** - Architecture evaluation and scoring
- **CONTEXT_MANAGEMENT_STRATEGY.md** - LLM context handling strategy
- **SELF_HEALING_DESIGN.md** - Self-healing mechanism design
- **SETTINGS_ARCHITECTURE.md** - Settings/configuration architecture
- **USER_MANAGEMENT_STRATEGY.md** - User management approach

### Guides (`guides/`)
Internal operational guides:
- **DOCS_SETUP_INSTRUCTIONS.md** - How to set up the docs site
- **IMPLEMENTATION_STATUS.md** - Current implementation status tracker
- **TEST_REPORT.md** - Test coverage and quality reports

### Deployment (`deployment/`)
Internal deployment guides:
- **DEPLOYMENT_CHECKLIST.md** - Deployment verification checklist
- **RAILWAY_DEPLOYMENT.md** - Railway-specific deployment

### Strategy (`strategy/`)
Internal business and competitive strategy:
- **ARGUS_SALES_BATTLECARD.md** - Sales enablement
- **ARGUS_STRATEGIC_KNOWLEDGE_BASE.md** - Strategic knowledge
- **COMPETITIVE_ANALYSIS*.md** - Competitor analysis

### Assets (`assets/`)
Images, diagrams, and other static assets.

## Contributing

When adding internal documentation:
1. Place files in the appropriate subdirectory
2. Use descriptive filenames with SCREAMING_SNAKE_CASE
3. Mark sensitive competitive information clearly
4. Update this README if adding new sections

For public documentation contributions, submit PRs to [argus-docs](https://github.com/RaphaEnterprises-AI/argus-docs).
